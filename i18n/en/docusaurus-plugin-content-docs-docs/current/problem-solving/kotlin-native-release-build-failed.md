# Ошибка Release сборки Kotlin/Native

Если у вас происходит ошибка компиляции только Release сборок Kotlin/Native, то возможно здесь вы найдете помощь.

## Ошибка из-за использования несовместимых версий kotlin и serialization

:::tip Быстрый ответ

При ошибке компиляции `Compilation failed: null` проверьте соответствует ли версия Kotlin той, которая нужна для используемой версии библиотеки kotlin-serialization.

:::

Библиотека [kotlinx-serialization](https://github.com/Kotlin/kotlinx.serialization) состоит из нескольких частей - во первых это рантайм библиотека, которую мы подключаем к проекту как обычную зависимость. И второе - компиляторный плагин, который подключается через подключение gradle плагина `org.jetbrains.kotlin.plugin.serialization`. В случаях когда рантайм библиотека и компиляторный плагин не совместимы, можно получить непонятную ошибку.

Чтобы найти причину пришлось постепенно отключать модули от компиляции, а как поняли в каком модуле ошибка компиляции происходит - начали в этом модуле комментировать классы, пока не обнаружили что ошибка происходит при наличии в коде классов с кастомными `KSerializer`'ами. Узнав это, стало понятно что дело как-то связано с kotlinx.serialization и проверили версию. Как оказалось у нас была подключена версия плагина такая же, как котлина - 1.4.31. А версия библиотеки была 1.2.1. Начиная с версии 1.2.0 библиотека компилировалась kotlin 1.5.0 и требовала соответствующе версию котлина в проекте 1.5.0. Мы поставили последнюю версию с kotlin 1.4 - это оказалась версия библиотеки 1.1.0, и проблема была решена - сборка kotlin/native проходила успешно и для android и для ios.
